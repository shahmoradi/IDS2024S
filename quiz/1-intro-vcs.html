This quiz is open-lecture, however, you are not allowed to consult other resources or people.
Despite the length of the quiz, many of the answers are brief, sometimes a simple YES or NO, so do not elaborate excessively on a particular question. Instead, try to answer all question briefly and correctly.

You have the entire class time to submit your answers for this quiz.
Ensure you stage and commit all your answers to your GitHub account in the right location for quiz 1.
GitHub automatically stamps all your submissions with time up to seconds.
Late submissions as evidenced by the GitHub stamps for the commit time will not count.
So, ensure to submit your work before the official end of class time.



  1. Create a README.md within a subfolder named 1 within the quiz folder that you have already created in your GitHub project as part of homework 1. This README.md file will contain the rest of your answers to the following questions.
    Once you write your answers to all questions in this file, commit this file to your GitHub project before quiz ends at the end of the class time.
    Tips for quiz submission:
    You can take two different approaches to creating the README.md file for this quiz that will reside in the quiz/1/ of your IDS2024S GitHub project.
    • You can create the quiz/1/README.md file on your GitHub project page account in your browser.
      This is the recommended approach if you still do not feel comfortable using the Git Bash command line.
      WARNING
      In this case, just like how you created your README.md file for the front page of your GitHub project, make sure to commit all your changes to your quiz README file before leaving the page.
      If you fail to commit the quiz README file (containing your quiz answers) before leaving the edit page, you will lose all your answers and I will have nothing to grade.
      If you are in doubt, get help from Nicholas or me.
    • You can make the ./quiz/1/README.md file in the clone of your IDS2024S project that we created in class on Wednesday.
      Then add all your answers in the file, then stage and commit the file to your project on the Git Bash command line, then push the changes to your project on GitHub just like how we did it in class on Wednesday.
      This is the recommended approach if you already feel comfortable using the Git Bash command line.
